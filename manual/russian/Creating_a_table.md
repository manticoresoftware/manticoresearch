# Создание таблицы
<!-- proofread -->

Manticore Search предлагает несколько способов создания таблиц, каждый из которых предназначен для различных вариантов использования и требований. В этом разделе рассматриваются основные подходы и их конкретные конфигурации.

## Типы таблиц

Manticore поддерживает различные типы таблиц:

- **Таблицы реального времени** — для обновления в реальном времени и мгновенного поиска
- **Обычные таблицы** — для статических данных, не требующих частых обновлений
- **Таблицы Percolate** — для хранения и сопоставления поисковых запросов
- **Таблицы-шаблоны** — для создания шаблонов таблиц
- **Распределённые таблицы** — для масштабирования на несколько узлов

## Ключевые компоненты

При создании таблицы необходимо учесть:

1. **Типы данных** — понимание доступных типов данных и вариантов их хранения
2. **Обработка естественного языка и токенизация** — настройка обработки и индексации текстовых данных
3. **Структура таблицы** — определение схемы и параметров для вашего конкретного варианта использования

## Начало работы

Чтобы создать таблицу, необходимо:

1. Выбрать подходящий тип таблицы для вашего варианта использования
2. Определить схему таблицы с необходимыми полями и Атрибуты
3. Настройте необходимые параметры в соответствии с вашими требованиями

Подробную информацию о каждом аспекте см. в следующих разделах:

- [Типы данных](Creating_a_table/Data_types.md) – Информация о поддерживаемых типах данных и вариантах хранения
- [Создание локальной таблицы](Creating_a_table/Local_tables.md) – Подробное руководство по созданию различных типов локальных таблиц
- [Обработка естественного языка и токенизация](Creating_a_table/NLP_and_tokenization/Data_tokenization.md) – Настройка обработки текста и индексации
- [Создание распределенной таблицы](Creating_a_table/Creating_a_distributed_table/Creating_a_distributed_table.md) – Настройка распределенных таблиц для масштабирования

В каждом из этих разделов представлена подробная информация о конкретных аспектах создания таблиц в Manticore Search.
