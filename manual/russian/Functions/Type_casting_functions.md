# Функции приведения типов

Приведение типов включает три основных действия: преобразование, переосмысление и продвижение.
* **Преобразование**. Это процесс изменения типа данных значения на другой тип данных. Это требует дополнительных вычислений и выполняется исключительно функцией `TO_STRING()`.
* **Переосмысление**. Это подразумевает обработку двоичных данных, представляющих значение, как если бы они имели другой тип данных, без фактического изменения исходных данных. Это обрабатывается функцией `SINT()`, не требует дополнительных вычислений; вместо этого просто переосмысляет существующие данные.
* **Продвижение**. Это процесс преобразования значения в "более крупный" или более точный тип данных. Не требует дополнительных вычислений; просто запрашивает, чтобы аргумент возвратил значение другого типа. Только поля JSON и несколько других функций могут продвигать свои значения до целочисленных. Если аргумент не может вернуть значение другого типа, продвижение завершится неудачей. Например, функция `TIMEDIFF()` обычно возвращает строку, но может также вернуть число. Следовательно, `BIGINT(TIMEDIFF(1,2))` выполнится успешно, заставляя `TIMEDIFF()` выдать целочисленное значение. В то же время `DATE_FORMAT()` возвращает только строки и не может выдать число, поэтому `BIGINT(DATE_FORMAT(...))` не будет выполнено.

### BIGINT()

Эта функция продвигает целочисленный аргумент к 64-битному типу, оставляя плавающие аргументы без изменений. Она предназначена для обеспечения вычисления некоторых выражений (например, `a*b`) в 64-битном режиме, даже если все аргументы 32-битные.

### DOUBLE()

Функция `DOUBLE()` продвигает свой аргумент к типу с плавающей точкой. Это предназначено для принудительного вычисления числовых полей JSON.

### INTEGER()

Функция `INTEGER()` продвигает свой аргумент к знаковому 64-битному типу. Это предназначено для принудительного вычисления числовых полей JSON.

### TO_STRING()

Эта функция принудительно преобразует свой аргумент в строковый тип.

### UINT()

Функция `UINT()` продвигает свой аргумент к 32-битному беззнаковому целочисленному типу.

### UINT64()

Функция `UINT64()` продвигает свой аргумент к 64-битному беззнаковому целочисленному типу.

### SINT()

Функция `SINT()` принудительно переосмысливает свой 32-битный беззнаковый целочисленный аргумент как знаковый и расширяет его до 64-битного типа (так как 32-битный тип беззнаковый). Например, 1-2 обычно оценивается как 4294967295, но `SINT(1-2)` оценивается как -1.
