# Профиль запроса

<!-- example SHOW PROFILE -->

Оператор SQL `SHOW PROFILE` и опция JSON-интерфейса `"profile": true` предоставляют детальный профиль выполнения выполненного запроса. В случае SQL, профилирование должно быть включено в текущей сессии **до** запуска оператора, который необходимо профилировать. Это можно сделать с помощью оператора `SET profiling=1`. По умолчанию профилирование отключено, чтобы избежать возможных ухудшений производительности, и если оно не включено, профиль будет пустым.

Каждый результат профилирования включает следующие поля:
* Столбец `Status` кратко описывает конкретное состояние, в котором было затрачено время. Подробнее см. ниже.
* Столбец `Duration` показывает время по настенным часам, в секундах.
* Столбец `Switches` отображает количество раз, когда движок запроса переходил в указанное состояние. Это лишь логические переключения состояния движка и **не** являются переключениями контекста на уровне ОС или вызовами функций (хотя некоторые секции могут соответствовать вызовам функций), и они **не** имеют прямого влияния на производительность. В некотором смысле, количество переключений — это просто количество срабатываний соответствующих точек инструментирования.
* Столбец `Percent` показывает процент времени, проведённого в этом состоянии.

Состояния в профиле возвращаются в предопределённом порядке, который примерно соответствует (но **не** совпадает полностью) с фактическим порядком запроса.

Список состояний может (и будет) меняться со временем по мере уточнения состояний. Вот краткое описание текущих профилируемых состояний.

* `unknown`: универсальное состояние по умолчанию. Показывает время, затраченное на код, который ещё не был инструментирован, или мелкие другие задачи, которые не подходят ни под одно другое состояние, но слишком малы, чтобы выделять отдельное состояние.
* `net_read`: чтение запроса из сети (т.е. от приложения).
* `io`: время ввода-вывода файлов в общем виде.
* `dist_connect`: подключение к удалённым агентам в случае распределённой таблицы.
* `sql_parse`: разбор синтаксиса SQL.
* `dict_setup`: настройка словаря и токенизатора.
* `parse`: разбор синтаксиса полнотекстового запроса.
* `transforms`: преобразования полнотекстового запроса (подстановки, расширения, упрощения и т.п.).
* `init`: инициализация оценки запроса.
* `open`: открытие файлов таблиц.
* `read_docs`: время ввода-вывода, затраченное на чтение списков документов.
* `read_hits`: время ввода-вывода, затраченное на чтение позиций ключевых слов.
* `get_docs`: вычисление соответствующих документов.
* `get_hits`: вычисление соответствующих позиций.
* `filter`: фильтрация полнотекстовых совпадений.
* `rank`: вычисление релевантного ранга.
* `sort`: сортировка совпадений.
* `finalize`: окончательная обработка результирующего набора поиска по таблице (выражения на последнем этапе и т.п.).
* `dist_wait`: ожидание удалённых результатов от агентов в случае распределённой таблицы.
* `aggregate`: агрегация нескольких наборов результатов.
* `net_write`: запись результирующего набора в сеть.

##### SQL:
<!-- request SQL -->

```sql
SET profiling=1;

SELECT id FROM forum WHERE MATCH('the best') LIMIT 1;

SHOW PROFILE;
```
