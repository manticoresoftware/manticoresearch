# Режим только для чтения

Режим только для чтения для подключения отключает любые модификации таблиц или глобальные изменения. Следовательно, запросы типа `create`, `drop`, различные виды `alter`, `attach`, `optimize`, а также запросы на изменение данных, такие как `insert`, `replace`, `delete`, `update` и другие, будут отклонены. Изменение настроек уровня демона с помощью `SET GLOBAL` также невозможно в этом режиме.

Однако вы по-прежнему можете выполнять все операции поиска, генерировать сниппеты и запускать запросы `CALL PQ`. Кроме того, вы можете изменять локальные (на уровне подключения) настройки.

Чтобы проверить, находится ли ваше текущее подключение в режиме только для чтения или нет, выполните оператор `show variables like 'session_read_only'`. Значение `1` указывает на режим только для чтения, а `0` означает, что режим не активен (обычный).

## Активация

Режим только для чтения может быть включен на 3 различных уровнях:

1. Для слушателя, добавив суффикс `_readonly` к директиве [listen](../Server_settings/Searchd.md#listen)
2. Для сессии, выполнив `SET ro=1`
3. На уровне демона, установив [`read_only = 1`](../Server_settings/Searchd.md#read_only) в секции `searchd`

Когда включен `searchd.read_only=1`, все новые клиентские подключения автоматически начинают работу в режиме только для чтения. Этот режим предназначен для развертываний, где Manticore не должен изменять хранилище таблиц, например, при обслуживании обычных таблиц с реплицированной файловой системы только для чтения. В этом режиме модифицирующие запросы отклоняются так же, как и на слушателе `_readonly`.

Глобальный `read_only` на уровне демона поддерживается только для обслуживания хранилища обычных таблиц. Он не может сочетаться с `data_dir` и не может использоваться в конфигурациях, объявляющих RT-таблицы или перколяционные таблицы.

## Деактивация

Если вы подключены к [VIP](../Server_settings/Searchd.md#listen) сокету, вы можете выполнить `SET ro=0` (даже если сокет, к которому вы подключены, был определен как доступный только для чтения в конфигурации, а не интерактивно). Это переключит подключение в обычный (не только для чтения) режим с разрешением всех модификаций.

Для стандартных (не VIP) подключений выход из режима только для чтения возможен только путем переподключения, если он был установлен интерактивно, или путем обновления файла конфигурации и перезапуска демона.

Если режим только для чтения был включен на уровне демона через `searchd.read_only=1`, `SET ro=0` не разрешен даже на VIP-подключениях. Отключите `read_only` в конфигурации и перезапустите демон, чтобы выйти из глобального режима только для чтения.
